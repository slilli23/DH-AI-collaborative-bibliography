# DH-AI Collaborative Bibliography

The Italian Observatory on AI and Digital Humanities (DH-AI) is a project launched
in late 2025 as a Special Interest Group (SIG) of the Italian Association for Digital
Humanities and Digital Culture (AIUCD). Among the SIG's core activities is the creation
of an open collaborative bibliography on AI and DH topics.

This repository contains supplementary material for the paper presented at the
EADH 2026 Conference.

## Contents

### Source
- `bibliography_export_2026-06-30.csv` — DH-AI Zotero bibliography (249 items).
- `tagging_guidelines.pdf` — instructions issued to contributors for the collaborative
  tagging task (in Italian).

### Derived data
- `language_attribution.csv` — language assigned to each item, joinable to the export on
  `Key`. Taken from the Zotero `Language` field where compiled (173 items) and inferred
  from title and abstract with `langdetect` where missing (76 items); the `source` column
  records which.
- `tag_mapping.csv` — tag normalisation table. Each of the 391 original tags is assigned
  an action (keep / merge / facet / drop) and, where applicable, a canonical concept. A *canonical concept* is the single reference form to which several original tags denoting
the same notion are reduced (`AI`, `Artifical intelligence` → `Artificial intelligence`;
`etica`, `ethics` → `Ethics`). Acronyms are expanded, non-English tags are rendered in
English, and typographical errors are corrected; the resulting form is recorded in
`canonical_concept`, while `graph_label` holds the abbreviation used to label the network
nodes. The
  draft was generated by an LLM (Claude Opus 4.8) and validated by hand. 
Four tags (`ricerca`, `scuola`, `università`, `tutti`) mark the intended audience of an
item rather than its subject matter. They are marked `facet` in the mapping and excluded
from the network, where they would link unrelated concepts by construction.
- `network_edgelist.csv`, `network_metrics.csv` — co-occurrence network of the normalised
  concepts: weighted edges, and per-node degree, strength, betweenness, eigenvector
  centrality and community.

### Figures
Each figure is provided as a PNG, previewed below, and as a self-contained interactive
HTML file. 
- `item_types.png` / `item_types.html` — distribution of items by type.
- `languages.png` / `languages.html` — distribution of items by language.
- `network.png` / `network.html` — concept co-occurrence network.

### Code
- `zotero_analysis.py` — reads the export; produces the two distribution charts,
  `language_attribution.csv`, and the raw item–tag table.
- `normalize_tags.py` — drafts and applies `tag_mapping.csv`; produces the normalised
  item–concept table.
- `network_analysis.py` — projects the concept co-occurrence network and computes its
  metrics.

## Reproducibility

Scripts: `analysis.py`, `normalize_tags.py`, `network_analysis.py`.
Concept co-occurrence is projected from the bipartite item–concept graph with Newman
weighting (an item indexing *n* concepts contributes 1/(*n*−1) to each pair). Concepts
occurring in fewer than 2 items are excluded. Communities are detected with the Louvain
algorithm (seed 42). Python 3.12; pandas, plotly, langdetect, networkx, pyvis.
